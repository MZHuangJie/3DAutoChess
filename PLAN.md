# Plan: 3D Auto Chess (金铲铲-like 自走棋)

## 1. Requirements Summary

- **Goal**: 在 Unity 6 中构建一个可运行的单人自走棋游戏（类似金铲铲 / TFT），玩家对战 AI，包含完整的准备阶段、自动战斗、经济、羁绊、装备、海克斯等核心体验。
- **Scope**:
  - **In-scope**:
    - 棋盘与英雄（纯色几何体占位）
    - 准备阶段 ↔ 战斗阶段循环
    - 商店、D牌、升级人口、买怪/卖怪
    - 英雄升星（3合1）
    - 羁绊系统（种族/职业组合加成）
    - 经济系统（金币、利息、连胜/连败）
    - 装备合成与穿戴
    - 英雄技能（简单自动释放）
    - 海克斯/天赋系统
    - 选秀轮、PvE野怪轮
    - 7个AI对手（可配置难度）
  - **Out-of-scope**:
    - 联网多人对战
    - 复杂3D美术、动画、特效（使用纯色几何体占位）
    - 音效与音乐
    - 存档/云同步
    - 移动端适配优化
- **Users / Context**: 单机PC玩家，在Unity Editor或Build中运行。
- **Tech Stack**:
  - Unity 6 (6000.0.x)
  - URP (已配置)
  - C# Scripts
  - Input System (已配置)
  - TextMeshPro (UGUI 内置)
  - 纯色 Cube/Sphere/Capsule 作为英雄/装备占位模型
- **Deliverables**:
  - 可运行的 Unity 场景
  - C# 脚本源代码
  - ScriptableObject 数据配置（英雄、羁绊、装备、海克斯）
  - UI 预制体
- **Constraints**:
  - 美术资源仅限于 Unity 基础几何体 + 纯色材质
  - AI 逻辑在本地运行，需注意性能
  - 分阶段交付，每阶段结束都有可玩版本

## 2. Selected Solution

- **Option**: 方案 A — 分阶段迭代实现
- **Justification**:
  - 自走棋系统耦合度高（经济→商店→羁绊→战斗全部相互影响），一次性搭建完整框架风险极大。
  - 分阶段迭代可以在每个里程碑得到一个可运行的版本，及时验证玩法，避免架构层面的大规模返工。
  - 后续阶段可以根据前面阶段的反馈灵活调整优先级（例如如果装备太复杂，可以先跳过）。

## 3. Implementation Steps

### 里程碑 1：MVP — 核心战斗循环可玩
*目标：玩家可以排兵布阵，与AI进行回合制自动战斗，能打完一局分出胜负。*

1. **项目结构搭建**
   - 创建文件夹结构：`Scripts/`, `Scripts/Core/`, `Scripts/Data/`, `Scripts/UI/`, `Scripts/AI/`, `Prefabs/`, `Materials/`, `ScriptableObjects/Heroes/`, `ScriptableObjects/Factions/`, `Scenes/`
   - 创建主游戏场景 `MainGame.unity`

2. **核心数据架构**
   - `HeroData` (ScriptableObject): 英雄基础属性（名称、生命值、攻击力、攻击速度、攻击范围、技能、费用、种族、职业）
   - `FactionData` (ScriptableObject): 羁绊定义（名称、各层级触发条件和效果）
   - `GameConfig` (ScriptableObject): 全局配置（棋盘大小、每回合时长、初始血量/金币等）

3. **棋盘与英雄实体**
   - `Chessboard` 管理器：8x8 准备棋盘 + 战斗棋盘（简化版可用 3x5 或 4x7）
   - `ChessPiece` MonoBehaviour：英雄实体，包含当前属性、位置、目标、状态机（Idle / Move / Attack / Dead）
   - 使用纯色 Capsule + 颜色区分不同英雄

4. **准备阶段（Preparation Phase）**
   - 玩家可以拖拽英雄在棋盘上移动（使用鼠标射线检测）
   - 简单的 Bench（候场区，可放额外英雄）
   - 阶段倒计时（如 30 秒），结束后自动进入战斗

5. **自动战斗系统（Combat Phase）**
   - 每个英雄每帧根据攻击速度决定行动
   - 寻敌逻辑：寻找最近敌人 → 如果在攻击范围内则攻击，否则移动
   - 伤害计算：攻击 - 防御（简化）
   - 一方全部死亡 → 战斗结束，根据存活英雄数量和星级结算伤害给输家

6. **回合循环与血量**
   - `GameLoopManager`：管理 Preparation → Combat → Result → Next Round
   - 玩家和 AI 都有 100 血量，战败扣血，归零则淘汰

7. **极简 AI（里程碑1版）**
   - 1 个 AI 对手
   - 随机从预设池中选取英雄
   - 随机摆放在其棋盘上
   - 无商店、无经济，每回合随机获得若干个英雄

8. **基础 UI**
   - 当前阶段/回合数显示
   - 玩家和 AI 血量显示
   - 倒计时显示
   - 战斗结果提示

### 里程碑 2：核心系统 — 商店、羁绊、经济
*目标：引入完整的商店D牌、人口升级、英雄升星、羁绊加成、经济循环。*

9. **商店系统**
   - `ShopManager`：根据当前等级从英雄池中按概率抽取 5 个英雄
   - 英雄池管理：每个英雄有总数限制（模拟卡池）
   - 操作：刷新商店（2金币）、购买英雄（花费费用）、锁定商店

10. **玩家经济与人口**
    - 金币获取：每回合基础收入 + 利息（每10金币存1利息，上限5）+ 连胜/连败奖励
    - 人口上限：初始3，可花费金币升级（经验值系统）
    - 英雄上限 = 人口上限，Bench 有独立上限（如 9 个）

11. **英雄升星（3合1）**
    - 当 Bench 或棋盘上有 3 个同名同星英雄时，自动合成一个更高星英雄
    - 3星为上限，星级提升属性倍数增长

12. **羁绊系统**
    - 战斗开始时根据场上存活英雄计算激活的羁绊
    - 羁绊效果应用：加攻击、加生命、加攻速、吸血等（简化版先做数值加成）
    - UI 显示当前激活羁绊

13. **AI 升级（7个AI）**
    - 增加到 7 个 AI 对手（标准8人局）
    - AI 有简单的决策逻辑：根据金币买怪、尝试凑羁绊、升级人口、调整站位
    - 玩家每回合随机与其中一个存活AI对战

14. **PvP 对战匹配**
    - 每回合随机匹配一个存活对手（不会连续两回合打同一个，除非只剩两人）
    - 镜像对手棋盘上的阵容进行对战

### 里程碑 3：进阶系统 — 装备、技能、AI强化
*目标：装备合成、英雄技能释放、更聪明的AI。*

15. **装备系统**
    - `EquipmentData` (ScriptableObject)：基础装备定义（属性加成或特效）
    - 合成配方：两个基础装备合成一个大件（如反曲弓+腰带 = 狂徒）
    - PvE 回合掉落散件，玩家拖拽装备到英雄身上穿戴

16. **英雄技能**
    - 每个英雄有蓝条（Mana），攻击和受击获得蓝量
    - 满蓝时释放技能（简化版：立即造成伤害/治疗/控制，无需复杂动画）
    - 技能配置在 `HeroData` 中（伤害类型、数值、范围、特效）

17. **PvE 野怪轮**
    - 特定回合（如 1-1, 1-2, 2-7, 3-7...）与野怪战斗
    - 野怪有固定属性，击败后掉落装备或金币

18. **AI 强化**
    - AI 尝试凑特定强力羁绊
    - AI 会根据对手调整站位（前后排）
    - AI 会合成装备并给核心C位

### 里程碑 4：完整体验 — 海克斯、选秀、UI完善
*目标：加入海克斯系统、选秀轮、完整的游戏流程和 polished UI。*

19. **海克斯/天赋系统**
    - 特定回合（1-4, 3-3, 5-2 等）提供 3 个海克斯选项
    - 海克斯效果：全局属性加成、特殊机制改变、经济加成等
    - `HexData` (ScriptableObject) 配置

20. **选秀轮（Carousel）**
    - 特定回合所有存活玩家围绕一个旋转的圆环选择带装备的英雄
    - 血量低的玩家优先选择
    - 简化实现：弹出选择界面，按血量排序依次选择

21. **UI 完善**
    - 完整的商店面板（英雄头像+费用+种族/职业标签）
    - 羁绊面板（显示当前激活和未激活羁绊）
    - 装备合成提示
    - 对战记录/伤害统计
    - 玩家排名面板

22. **游戏流程与结束**
    - 启动画面 → 加载 → 第1回合准备
    - 最后存活者获胜，显示排名和统计
    - 重新开始功能

## 5. Risks & Assumptions

- **Assumptions**:
  - Unity 6 URP 基础渲染功能正常工作，不需要额外配置
  - 纯色几何体（Cube/Capsule/Sphere）足以支撑 gameplay 验证
  - 性能上本地运行 8 组棋盘 + AI 计算在可接受范围内（可优化）

- **Risks**:
  - **Risk**: 自动战斗系统的寻路和攻击判定容易出现边界情况（如卡位、目标死亡后重新寻敌）
    - **Mitigation**: 使用简化的网格移动（不走 A*，直接直线移动），充分测试边缘情况
  - **Risk**: AI 逻辑如果太复杂会导致开发时间过长
    - **Mitigation**: AI 从随机行为开始，逐步增加规则，不追求完美最优解
  - **Risk**: 8人对战同时运行多组战斗可能导致性能问题
    - **Mitigation**: 实际战斗是玩家 vs 一个AI镜像，其他AI之间的对战用简化模拟（直接根据战力算伤害）
  - **Risk**: 系统间耦合度高，后期加功能可能需要重构前期代码
    - **Mitigation**: 前期设计好事件系统和数据驱动架构（ScriptableObject），减少直接依赖
